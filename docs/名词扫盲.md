1.分析文件.(主要包含以下几种格式文件)
(1)Gerber-RS274X :一般以"FSLAX*Y*"开头, 以"M02"结尾.
(2)Gerber-RS274D :以"G90"开头, 以"M02"结尾, 因没有包含图形定义内容, 需定义Aperture Wheel文件(包含Dcode, 形状, Size).
(3)ODB++ :一般后续名为"TGZ", 并且压缩包里面包含"steps"&"symbols"等文件夹.
(4)Excellon :一般以"T*C*"开头(如钻带头与钻带分开,需编辑), 以"M30"结尾.
(5)Netlist(IPC-D-356与Neutral) :"IPC"以'317'或'327'开头, "Neutral"以'P_ADDP'开头.
(6)HPGL(HPGL1与HPGL2) :"HPGL1"以'PA'或'PD'或'PU'开头, "HPGL2"以'@PJL SET'开头.
(7)IPC2581A / C   

2.Input文件到工作站.
(1)Gerber-RS274D需定义相关图形定义文件(Aperture Wheel).
(2)Excellon需定义相关钻孔大小(格式必须为T*C*).

Pad 指的是 焊盘 — PCB 上用来焊接元器件引脚的铜箔区域。

add_pad 是 Genesis 的 API 命令，在电路板上放置一个焊盘。参数含义：

  ┌──────────┬───────────────────────────────────────┐
  │   参数    |                 含义                  │
  ├──────────┼───────────────────────────────────────┤
  │ x, y     │ 焊盘坐标位置                            │
  ├──────────┼───────────────────────────────────────┤
  │ symbol   │ 焊盘形状（如 mot-xx、r200）              │
  ├──────────┼───────────────────────────────────────┤
  │ polarity │ positive = 铜箔，negative = 开窗/清除   │
  ├──────────┼───────────────────────────────────────┤
  │ angle    │ 旋转角度                               |
  ├──────────┼───────────────────────────────────────┤
  │ nx, ny   │ X/Y 方向重复数量 					   |
  ├──────────┼───────────────────────────────────────┤
  │ dx, dy   │ X/Y 方向间距                           |
  └──────────┴───────────────────────────────────────┘
 简单说就是把某个特定形状的焊盘放到板子的指定位置。

polarity：
Polarity（极性） 跟正反面没关系：
- positive = 有铜（就是实心的焊盘）
- negative = 没有铜（在那个区域挖空/清除铜箔）
正反面是由当前操作的 layer（层） 决定的，不是polarity。

开窗 也不是打洞：
- 开窗 = 在阻焊层（绿油）上开一个口，把铜皮露出来才能焊接
- 打洞/钻孔 = 用钻头在板子上钻出物理孔洞（via/through-hole）
两者不同：开窗是去掉绝缘涂层，钻孔是去掉基材本身。
